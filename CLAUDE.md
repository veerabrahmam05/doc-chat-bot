## High‑Level Architecture

- **Frontend** – a Next.js 16 app written in TypeScript. UI components live under `frontend/components/`. The app communicates with the backend via HTTP calls to the API (e.g., chat and upload endpoints).
- **Backend** – a FastAPI service (`backend/src/app.py`). It exposes two routers:
  - `POST /upload` – stores the uploaded file, extracts text (PDF only), chunks it, creates embeddings via **sentence‑transformers**, builds a FAISS index, and keeps the data in an in‑memory `documents_store` dict.
  - `POST /chat` – receives a document ID and a question, retrieves the stored vectors, performs a similarity search (FAISS), and forwards the relevant chunks to an LLM endpoint.
- **Embedding & Retrieval** – `src/services/embedding_service.py` handles chunking, embedding generation, and FAISS indexing/search.
- **LLM Integration** – `src/services/llm_service.py` calls an Ollama server (configured by `OLLAMA_URL` in `.env`) using the `gpt-oss:120b-cloud` model.
- **Configuration** – environment variables are loaded via `src/config/env.py` (`UPLOAD_DIR`, `OLLAMA_URL`). Ensure a `.env` file exists at the repository root.
- **In‑Memory Store** – `src/services/store.py` (not shown) provides the global `documents_store` dictionary that maps a UUID to the document’s chunks and FAISS index.

## Important Operational Details

- The backend creates the upload directory on first request if it does not exist (`UPLOAD_DIR`).
- Uploaded PDFs are processed with **PyPDF2**; other file types are rejected.
- Embeddings are computed with the **all‑MiniLM‑L6‑v2** SentenceTransformer model.
- FAISS uses a flat L2 index; similarity search returns the top‑3 most relevant chunks.
- The LLM prompt explicitly instructs the model to answer *only* using the provided context.

## Quick Reference for New Contributors

1. Clone the repo and run `npm install && pip install -r backend/uv.lock`.
2. Create a `.env` file with `UPLOAD_DIR=./uploads` and `OLLAMA_URL=http://localhost:11434/api/generate` (or your Ollama endpoint).
3. Start the backend (`uvicorn src.app:app --reload`) and frontend (`npm run dev`).
4. Use the UI to upload a PDF and ask questions; the flow follows the pipeline described above.
